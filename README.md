# Tourist Bureau

## Description

Консольное приложение для автоматизации работы туристического бюро.  
Поддерживает аутентификацию пользователей, CRUD-операции для маршрутов, автобусов и рейсов, а также формирование отчётов.  
Использует SQLite для хранения данных, CMake для сборки и GoogleTest для тестирования.

---

## Project Structure

```text
.
├── src/                  # Исходный код приложения
├── includes/             # Заголовочные файлы
├── tests/                # Unit-тесты (GoogleTest)
├── sql/                  # SQL-скрипт инициализации БД
├── data/                 # Каталог для временных файлов тестов
├── .github/workflows/    # CI (GitHub Actions)
├── bin/                  # Исполняемый файл
├── build/                # Директория сборки
└── CMakeLists.txt
```

## Installation

### Требования

- CMake 3.14+

```bash

```

## Testing

Запуск тестов:

```bash
ctest --test-dir build --output-on-failure
```

Тесты покрывают:
- работу с базой данных  
- аутентификацию  
- CRUD-операции  
- отчёты  

---

## Continuous Integration

Используется GitHub Actions для:
- сборки проекта  
- запуска тестов  

Запускается при `push` и `pull request`.

---

## Database Initialization

База данных создаётся автоматически при первом запуске.

- SQLite создаёт файл базы данных, если он отсутствует  
- Программа проверяет наличие таблиц  
- При отсутствии выполняется SQL-скрипт:

```text
sql/init.sql
```

Скрипт создаёт все таблицы и начальные данные.

### Тестовая база данных

```text
data/test.db
```

Используется только в тестах для изоляции данных.

---

## Notes

- Не требуется ручная настройка базы данных  
- Папка `data/` хранится в репозитории как каталог для временных файлов тестов  
- Во время выполнения тестов в ней могут создаваться `test.db` и другие служебные файлы  
- База данных инициализируется автоматически
